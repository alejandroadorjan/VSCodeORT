# Validate and Open the Pull Request

Before opening a pull request:

* run the build or compile check before tests
* run focused tests for the changed area
* run linting when relevant
* run `npm run valid-layers-check` for cross-layer imports
* manually verify the original scenario

In the pull request, link the issue, explain the change, include test steps, and keep the branch up to date with `main`.

After opening it, follow CI, respond to review, and understand that milestones and iteration plans affect when accepted work is scheduled.
