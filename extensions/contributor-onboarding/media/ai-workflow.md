# AI-Assisted Contributor Workflow

AI can help with onboarding when it follows the same contribution journey:

1. Summarize the issue and expected behavior.
2. Identify the likely architecture layer and owner area.
3. Search for existing patterns and nearby tests.
4. Propose a focused implementation plan.
5. Explain which validation commands are relevant.
6. Draft a pull request summary and manual test plan.

Use AI as a navigator, not as a replacement for project judgment. The contributor still owns architecture choices, validation, and review responses.
